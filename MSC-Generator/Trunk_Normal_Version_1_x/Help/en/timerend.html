TIMEREND - Diagram Element

Use the commands timerbegin and timerend to model a time measurement. The command timerend defines the end of the time measurement. There is a vertical line connecting the start and end points of the time measurement. Should the time measurement be displayed without a line, use the commands settimer, stoptimer and timeout.
Syntax
timerend: timerid, text, style
Parameters
timerid
The ID of the time measurement for which you define the end point.
text (optional)
The text that should be displayed at the start of the time measurement. If your text contains punctuation, you must put the complete text in quotes ("").
style (optional)
Sets the display mode for the time measurement end point.
Value Description
- The time measurement ends normally
* The time measurement is aborted
The default value for this parameter is "-".
Example
DiagramName: Time Measurements 1
DiagramStyle: uml
PageSize: 620,auto
PageMargins: 10,10,10,10

process: p1, "Process 1",,70
process: p2, "Process 2",,0,20

timerbegin: t1, p1, t1=now,l,o
msg: p1, p2, Message 1;
timerbegin: t2, p1, t2=now,l,n
timerbegin: t3, p2, t3,r,i
msg: p1, p2, Message 2;
;
timerbegin: t4, p2,,r,n
msg: p2, p1, Message 3
timerend: t1, tmax=t1+3s
timerend: t2, tmax=t2+4s;
;
timerbegin: t5, p1
msg: p1, p2, Message 4
timerend: t4,{0...4s};
;
msg: p2, p1, Message 5
timerend: p5
timerend: p3, {0...5s},*;
