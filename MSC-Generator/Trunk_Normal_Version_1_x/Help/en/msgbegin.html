MSGBEGIN - Diagram Element

Use the commands msgbegin and msgend if you want the message to span over a longer time interval (vertical offset) between sending and reception. A message represents communication from one actor or process to another one. There are three types of messages: asynchronous, synchronous and response messages. In SDL mode there is no difference between synchronous and asynchronous. Use the command msg if you want to display a message with no vertical offset in the diagram.
Syntax
msgbegin: messageid, sourceid, destinationid, message, style
Parameters
messageid
The ID of this message. Specify this ID in the command msgend.
sourceid
The ID of the lifeline which should send the message, or one of the following constants.
Value Description
ENV_LEFT The message comes from the left border of the diagram.
ENV_RIGHT The message comes from the right border of the diagram.
destinationid
The ID of the lifeline which should receive the message, or one of the following constants.
Value Description
ENV_LEFT The message goes to the left border of the diagram.
ENV_RIGHT The message goes to the right border of the diagram.
message
The text that should appear as the message. If your text contains punctuation, you must put the complete text in quotes ("").
style (optional)
Sets the type of message.
Value Description
+ asynchronous message
! synchronous message
(not supported in SDL diagrams)
* response
The default value for this parameter is "+".
Example
DiagramName: Messages
DiagramStyle: uml
PageSize: 600,auto
PageMargins: 10,10,10,10

process: p1, Process 1
process: p2, Process 2

found: p1, "found message","g1",l;
msgbegin: m1, p2, p1, "long transmission of a message"
linecomment: p2, "{t=now}",r;
msg: p1,p2, "asynchronous message";
msgend: m1
linecomment:p1, "{t+1s}";
msg: p1,p2, "synchronous message",!;
msg: p2, p1, "response",*;
lost: p2, "lost message", ,r
