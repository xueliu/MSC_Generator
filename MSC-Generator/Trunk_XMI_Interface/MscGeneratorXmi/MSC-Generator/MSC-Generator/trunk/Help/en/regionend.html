REGIONEND - Diagram Element

Use the command regionbegin and regionend to change the way the lifeline is displayed. The command regionend defines the end of the modified lifeline region. The lifeline of an instance can be displayed in activation, coregion or suspension mode.
Syntax
regionbegin: instanceid
Parameters
instanceid
The ID of the lifeline for which the display mode should be changed back to the previously set one.
Example
DiagramName: Regions
DiagramStyle: uml
PageSize: 600,auto
PageMargins: 10,10,10,10

process: p1, Process 1
process: p2, Process 2
process: p3, Process 3

regionbegin: p1, coregion
regionbegin: p2, coregion
regionbegin: p3, coregion;

msg: p1, p2, Message 1
regionbegin: p2, activation;
regionend: p2;

regionbegin: p3, activation
msg: p1, p3, Message 2;
regionend: p3;

regionend: p1
regionend: p2
regionend: p3;

msg: p2, p3, Message 3
regionbegin: p3, suspension;
;
;
regionend: p3;
