TIMERBEGIN - Diagram Element

Use the commands timerbegin and timerend to model a time measurement. The command timerbegin defines the end of the time measurement. There is a vertical line connecting the start and end points of the time measurement. Should the time measurement be displayed without a line, use the commands settimer, stoptimer and timeout.
Syntax
timerbegin: timerid, instanceid, text, orientation, style
Parameters
timerid
The ID of the time measurement. The ID is required for the command timerend in order to match it to the correct time measurement.
instanceid
The ID of the lifeline at which the time measurement should be displayed.
text (optional)
The text that should be displayed at the end of the time measurement. If your text contains punctuation, you must put the complete text in quotes ("").
orientation (optional)
Sets the position of the time measurement in respect to the lifeline specified in the parameter instanceid.
Value Description
l The measurement will be displayed to the left of the lifeline
r The measurement will be displayed to the right of the lifeline
Der Standardwert für den Parameter ist "l".
style (optional)
Sets the display style of the timer.
Value Description
n The measurement will be displayed close to the lifeline
o The measurement will be displayed far from the lifeline, the text is from the outside
i The measurement will be displayed far from the lifeline, the text is from the inside
The default value for this parameter is "n".
Example
DiagramName: Time Measurements 1
DiagramStyle: uml
PageSize: 620,auto
PageMargins: 10,10,10,10

process: p1, "Process 1",,70
process: p2, "Process 2",,0,20

timerbegin: t1, p1, t1=now,l,o
msg: p1, p2, Message 1;
timerbegin: t2, p1, t2=now,l,n
timerbegin: t3, p2, t3,r,i
msg: p1, p2, Message 2;
;
timerbegin: t4, p2,,r,n
msg: p2, p1, Message 3
timerend: t1, tmax=t1+3s
timerend: t2, tmax=t2+4s;
;
timerbegin: t5, p1
msg: p1, p2, Message 4
timerend: t4,{0...4s};
;
msg: p2, p1, Message 5
timerend: p5
timerend: p3, {0...5s},*;
