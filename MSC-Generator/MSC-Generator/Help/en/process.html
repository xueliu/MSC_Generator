PROCESS - Diagram Element

The command process adds a new process (or object) to the diagram. The parameter id is used to assign a unique ID for the lifeline of the process. The parameters name and description define the name and an optional description of the process. By default actors and processes are equally distributed horizontally in a diagram. Sometimes it might be necessary to have more than the default space between an actor and its neighbors. The space to the left and to the right can be control with the arguments left und right.
Syntax
process: id, name, description, left, right
Parameters
id
The ID of the lifeline. Could be any alphanumeric identifier, which uniquely identifies the lifeline of the process. This ID is not visible in the diagram.
name
The name of the process. Some text that defines the process's name. The name is displayed in the diagram right below the process symbol. If your text contains punctuation, you must put the complete text in quotes ("").
description (optional)
The description of the process. Some text that describes the process. This text appears in the diagram right above the process symbol. If your text contains punctuation, you must put the complete text in quotes ("").
left (optional)
Additional place to the left of the process. The value is always in pixel.
right (optional)
Additional place to the right of the process. The value is always in pixel.
Example
DiagramName: Instances
DiagramStyle: uml
PageSize: 600,auto
PageMargins: 10,10,10,10

actor: actor1, Actor 1, Actor Description
process: process1, Process 1, Description, 70, 0
actor: actor2, Actor 2
