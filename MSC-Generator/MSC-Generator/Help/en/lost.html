LOST - Diagram Element

Use the command lost to display a lost message in the diagram. A lost message is a message whose receiver is either unknown or irrelevant, or the message is never received. Therefore the receiver will is not displayed on the diagram.
Syntax
lost: instanceid, message, gate, orientation
Parameters
instanceid
The ID of the lifeline which should receive the message.
message
The text that should appear as the message. If your text contains punctuation, you must put the complete text in quotes ("").
gate (optional)
The text that should be displayed below the found message. If your text contains punctuation, you must put the complete text in quotes ("").
orientation (optional)
Sets the position of the found message in respect to the lifeline specified in the parameter instanceid.
Value Description
l The found message will be displayed to the left of the lifeline.
r The found message will be displayed to the right of the lifeline.
The default value for this parameter is "l".
Example
DiagramName: Messages
DiagramStyle: uml
PageSize: 600,auto
PageMargins: 10,10,10,10

process: p1, Process 1
process: p2, Process 2

found: p1, "found message","g1",l;
msgbegin: m1, p2, p1, "long transmission of a message"
linecomment: p2, "{t=now}",r;
msg: p1,p2, "asynchronous message";
msgend: m1
linecomment:p1, "{t+1s}";
msg: p1,p2, "synchronous message",!;
msg: p2, p1, "response",*;
lost: p2, "lost message", ,r
