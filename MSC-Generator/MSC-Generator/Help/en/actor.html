ACTOR - Diagram element

The command actor adds an actor to the diagram. An actor represents a role in the context of the system you are modeling. The actor can influence the system and can be influenced by the system. An actor could stand for a person, group of people, a sensor, another system, etc. The parameter id is used to assign a unique ID for the lifeline of the actor. The parameters name and description define the name and an optional description of the actor. By default actors and processes are equally distributed horizontally in a diagram. Sometimes it might be necessary to have more than the default space between an actor and its neighbors. The space to the left and to the right can be control with the arguments left und right.
Syntax
actor: id, name, description, left, right
Parameters
id
The ID of the lifeline. Could be any alphanumeric identifier, which uniquely identifies the lifeline of the actor. This ID is not visible in the diagram.
name
The name of the actor. Some text that defines the actor's name. The name is displayed in the diagram right below the actor symbol. If your text contains punctuation, you must put the complete text in quotes ("").
description (optional)
The description of the actor. Some text that describes the actor. This text appears in the diagram right above the actor symbol. If your text contains punctuation, you must put the complete text in quotes ("").
left (optional)
Additional place to the left of the actor. The value is always in pixel.
right (optional)
Additional place to the right of the actor. The value is always in pixel.
Example
DiagramName: Instances
DiagramStyle: uml
PageSize: 600,auto
PageMargins: 10,10,10,10

actor: actor1, Actor 1, Actor Description
process: process1, Process 1, Description, 70, 0
actor: actor2, Actor 2
