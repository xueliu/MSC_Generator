MEASUREBEGIN - Diagram Element

Use the commands measurebegin and measureend to model a measurement. The command measurebegin defines the start of the measurement. The beginning and the end of a measurement are connected with a line on the generated diagram. It is not possible to nest multiple measurements on the same lifeline. For this purpose use the commands measurestart und measurestop which are displayed without a line connecting the start and the end.
Syntax
measurebegin: instanceid, text, orientation, style
Parameters
instanceid
The ID of the lifeline on which the measurement should be displayed.
text (optional)
The text that should be displayed at the beginning of the measurement. If your text contains punctuation, you must put the complete text in quotes ("").
orientation (optional)
Sets the position of the measurement in respect to the lifeline specified in the parameter instanceid.
Value Description
l The measurement will be displayed to the left of the lifeline
r The measurement will be displayed to the right of the lifeline
The default value for this parameter is "l".
style (optional)
Sets the display mode for the measurement.
Value Description
- The measurement arrows will be displayed below the horizontal measurement lines.
* The measurement arrows will be displayed above the horizontal measurement lines.
The default parameter for this parameter is "-".
Example
DiagramName: Measurements
DiagramStyle: uml
PageSize: 550,auto
PageMargins: 10,10,10,10
Left: 50
Right: 50

process: p1, "Process 1"
process: p2, "Process 2"

measurebegin: p1, M1
measurebegin: p2, M2, r, *
msg: p1, p2, Message 1;
;
;
msg: p2, p1, Message 2
measureend: p1
measureend: p2, end;
