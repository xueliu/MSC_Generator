DUMMYPROCESS - Diagram Element

The command dummyprocess adds a placeholder for a process (or object) to the diagram. A dummy process is invisible until it is created using the command create. The parameter id is used to assign a unique ID for the lifeline of the process. The parameters name and description define the name and an optional description of the process. By default actors and processes are equally distributed horizontally in a diagram. Sometimes it might be necessary to have more than the default space between an actor and its neighbors. The space to the left and to the right can be control with the arguments left und right.
Syntax
dummyprocess: id, left, right
Parameters
id
The ID of the lifeline. Could be any alphanumeric identifier, which uniquely identifies the lifeline of the process. This ID is not visible in the diagram.
left (optional)
Additional place to the left of the process. The value is always in pixel.
right (optional)
Additional place to the right of the process. The value is always in pixel.
Example
DiagramName: Instances
DiagramStyle: uml
PageSize: auto
PageMargins: 10,10,10,10

process: p1, Process 1
dummyprocess: p2, 50,0
process: p3, "Process 3"

create: p1, p2, neu, Process 2, new Process;
msg: p2, p3, end;
stop: p3;
