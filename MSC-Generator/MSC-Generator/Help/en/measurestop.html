MEASURESTOP - Diagram Element

Use the command measurestart to model a measurement in a diagram. The start and stop points are not connected with a line when displayed on the diagram. The end of the measurement is defined using the command measurestart.
Syntax
measurestop: instanceid, gate, text, orientation, style
Parameters
instanceid
The ID of the lifeline at which the measurement should be displayed.
gate (optional)
The text that should be displayed at the end gate of the measurement. If your text contains punctuation, you must put the complete text in quotes ("").
text (optional)
The text that should be displayed at the end of the measurement. If your text contains punctuation, you must put the complete text in quotes ("").
orientation (optional)
Sets the position of the measurement in respect to the lifeline specified in the parameter instanceid.
Value Description
l The measurement will be displayed to the left of the lifeline
r The measurement will be displayed to the right of the lifeline
The default value for this parameter is "l".
style (optional)
Sets the display mode for the measurement.
Value Description
- The measurement arrows will be displayed below the horizontal measurement lines.
* The measurement arrows will be displayed above the horizontal measurement lines.
The default parameter for this parameter is "-".
Example
DiagramName: Measurements 2
DiagramStyle: uml
PageSize: 550,auto
PageMargins: 10,10,10,10
Left: 50
Right: 50

process: p1, "Process 1"
process: p2, "Process 2"

measurestart: p1, m1, M1
measurebegin: p2, m2, M2, r, *
msg: p1, p2, Message 1;
;
;
;
msg: p2, p1, Message 2
measurestop: p1, m1
measurestop: p2, m2, end, r, *;
